## Detalhes de Implementação

### Ordem dos plugins importa?

Para os 3 plugins deste minigame, **ordem não é crítica** porque:
- `ButtonPicture` e `TextPicture` são independentes (não se referenciam).
- `Jhonny_RaceHelper` apenas estende `Input.keyMapper` no boot — não há dependência de carregamento entre eles.

No entanto, por convenção, plugins utilitários devem vir **antes** de plugins de gameplay. Ordem sugerida:
1. `TextPicture.js`
2. `ButtonPicture.js`
3. `Jhonny_RaceHelper.js`

### Parâmetros do `Jhonny_RaceHelper`

O plugin tem 1 parâmetro: `EnableDebugLogs` (boolean, default `true`). Para o protótipo, manter `true` (gera logs em F12 para debug). Em produção, mudar para `false`.

### Verificação do `plugins.js`

Após salvar no MZ Editor, o arquivo `Jhonny/js/plugins.js` deve conter entradas como:

```javascript
// Generated by RPG Maker MZ.
var $plugins = [
    {"name":"TextPicture","status":true,"description":"...","parameters":{}},
    {"name":"ButtonPicture","status":true,"description":"...","parameters":{}},
    {"name":"Jhonny_RaceHelper","status":true,"description":"Helpers para o minigame de Corrida (sem lógica de jogo).","parameters":{"EnableDebugLogs":"true"}}
];
```

O campo `"status":true` indica que o plugin está ativo.

### O que pode dar errado

| Sintoma | Causa | Solução |
|---------|-------|---------|
| Plugin não aparece na lista do Plugin Manager | Arquivo não está em `js/plugins/` | Confirmar que `Jhonny/js/plugins/Jhonny_RaceHelper.js` existe (task 1.2) |
| Erro "Plugin is not a function" no boot | Sintaxe JS inválida no plugin | Rodar `node -c Jhonny/js/plugins/Jhonny_RaceHelper.js` |
| Log não aparece no console | Plugin está OFF no Plugin Manager | Confirmar checkbox ON no Plugin Manager |
| `JhonnyRace` é `undefined` no console | Plugin não carregou | Verificar ordem do `plugins.js` e erros anteriores no console |
