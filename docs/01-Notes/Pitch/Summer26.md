# Summer Tavern Games (Tavern Jam)

> Resumo estruturado do vídeo de abertura da GameJam que a **Coreto** vai participar. Construído para leitura rápida por IA/agentes — cada seção é autocontida.

## TL;DR (contexto mínimo)

- **O que é:** GameJam casual, porém ranqueada, organizada pela comunidade Tavern.
- **Time:** Coreto.
- **Tema da temporada:** *"Let chance decide"* — mecânicas baseadas em acaso/RNG/dado.
- **Regra crítica (hard block):** **Proibido usar IA**. Uso de IA = desclassificação imediata.
- **Entregáveis obrigatórios:** jogo + vídeo de gameplay (YouTube/Vimeo/etc.).
- **Diferencial de votação:** jogos **web-playable** recebem consideravelmente mais votos que binários para download.

---

## Formato e fluxo de votação

1. Times/solo submetem o jogo dentro do prazo.
2. A comunidade vota no **top 5** jogos.
3. Os juízes + host analisam esses 5 para definir:
   - Vencedores por **categoria**.
   - Vencedor **geral**.

Filosofia: é uma jam *casual* — o objetivo declarado é se divertir, experimentar e aprender. Mas a ranqueamento é real.

---

## Critérios de julgamento (usar como spec de qualidade)

| Critério      | O que os juízes avaliam |
|---------------|-------------------------|
| **Design**    | Facilidade de entender e pegar o jogo (onboarding/clareza). |
| **Gameplay**  | Quão divertido e engajador; "dá vontade de continuar jogando?". |
| **Creativity**| Interpretação do tema — pode ser literal, flexível ou subvertida. |
| **Polish**    | Coesão visual/sonora/funcional; sensação de produto finalizado. |

## Regras

### O que é permitido
- Trabalhar **solo ou em equipe**.
- Usar assets de terceiros (música, arte, código) **desde que todos sejam referenciados** na submissão.

### O que é proibido
- **Qualquer uso de IA** (arte, código, música, texto, etc.). Motivo declarado: a jam é sobre criatividade humana; IA "derrota o ponto" segundo a comunidade. → **Desclassificação.**

### Recomendações fortes (impactam votação)
- **Web-playable é fortemente preferido.** Jogos só para download recebem menos votos por questões de segurança/fricção.
- **Incluir vídeo de gameplay** obrigatório — protege contra crashes, problemas de runtime, dificuldade alta que impeça juízes de terminar.

---

## Tema: *"Let chance decide"*

Interpretação oficial e exemplos dados pelo host:

- Explorar como o **acaso** pode moldar a jogabilidade ou a narrativa.
- Referências citadas: dados, **D&D**, jogos de tabuleiro, RPGs que usam RNG para direcionar a história.
- Pode ser **literal, flexível ou subvertido** — desde que exista **elemento de chance impactando gameplay** de alguma forma.
- Pergunta-guia deixada no ar: *"o acaso dita para onde você vai... ou será que não?"* (sugere espaço para subverter a própria noção de aleatoriedade).

## Implicações táticas para a Coreto

Derivadas do regulamento, para uso no planejamento:

1. **Stack/engine precisa exportar para web** (HTML5/WASM) — não é opcional se queremos votos.
2. **Zero IA no pipeline** — qualquer ferramenta generativa (mesmo para placeholders) está fora.
3. **Design deve ter onboarding curto** — critério explícito de "Design".
4. **Áudio é vetor de diferencial** — TK julga isso especificamente; investir em audio criativo é vantajoso.
4. **Direção de arte coerente** vale mais que detalhe técnico — Pepe valoriza identidade/charme.
5. **Inovação narrativa/mecânica** é o que Ahmad busca — favorece ideias arriscadas com potencial.
6. **Chance deve ser central**, não decorativo — o tema pede que RNG/acaso impacte gameplay de forma sentida.
7. **Gravar gameplay desde cedo** — vídeo é entregável obrigatório e seguro contra falhas de runtime na avaliação.
8. **Referenciar 100% dos assets de terceiros** — omissão pode ser lida como violação.

## Glossário rápido

- **RNG** — Random Number Generation; aleatoriedade computacional.
- **GDD** — Game Design Document.
- **Web-playable** — jogo que roda no navegador (sem instalação).
